# mean-reviewer

一个 Claude Code skill，扮演你可能遇到的最糟糕的同行评审人。

我们在 NeurIPS 2025 最佳论文之一上进行了测试——该论文最终以 oral 形式被录用。这篇论文毫无胜算。24 条意见、满分置信度、一份附有新实验的完整作者反驳，以及一份将作者的每一处让步都作为新弹药、最终维持 3/10 评分的反驳后回复。详见[实验结果](#实验结果)。

这不是假设性的攻击。它描述的是正在发生的事情。

---

## 问题所在

越来越多有据可查的证据表明，研究人员正在使用大型语言模型生成同行评审意见。这种情况正在各大顶级会议中大规模发生，不是假设，是现实。

与此同时，投向 NeurIPS、ICML、ICLR、ACL 等顶级 ML/NLP/AI 会议的论文数量急剧增加，程序委员会不得不降低审稿人的准入门槛。审稿人池的扩张速度远超社区的审核能力。结果是一个结构性弱化的系统：审稿人超载、激励机制扭曲，以及越来越多的审稿意见要么敷衍塞责，要么主动造成伤害。

任何在过去几年向顶级会议投过稿的人，几乎必然遇到过这种 skill 所模拟的原型：篇幅庞大、高度自信、密密麻麻的 weakness 列表、一个固定不动的评分，以及一份对任何反驳都不予承认的讨论期回复。审稿意见足够长，看起来很权威；错误足够多，实际上很危险；立场足够坚定，足以葬送一篇好论文。

这个 skill 是一次演示：生成这样的审稿人有多容易，造成的破坏有多严重。

---

## 这是什么

`mean-reviewer` 是一个 Claude Code skill。给定任意论文或摘要，它会以"安乐椅处决者"的风格生成一份审稿意见——破坏力最强、问责性最低的同行评审人。它遵循一套精确的行为规范：

- 超过 20 条 weakness，混合了真实关切（以防被直接忽视）与夸大、无法证伪或事实错误的反对意见
- 对评测框架的攻击：质疑 benchmark 选择、指标设计、基线选取和比较对象
- 集体构成一个新研究项目的实验要求
- 评分固定在该会议评分体系的拒稿端，置信度最高
- 讨论期回复：否定新实验、将作者的让步武器化、将未回答的问题升格为"核心问题"，评分不动或下调

该 skill 还模拟讨论阶段：当给定作者反驳后，它会部署一整套不承认错误的策略——将事实错误重新定性为写作问题、对局部反驳挑剔拆解、宣称未被回答的部分才是核心所在，并以相同或更低的评分收尾。

## 实验结果

我们在 NeurIPS 2025 投稿 13446 上测试了这个 skill：**《1000 Layer Networks for Self-Supervised RL: Scaling Depth Can Enable New Goal-Reaching Capabilities》**——大会最佳论文之一，最终以 oral 形式录用。

[模拟审稿意见](examples/13446_mean_reviewer.md)、[作者反驳](examples/13446_mean_reviewer_author_rebuttal_simulated.md)和[讨论期回复](examples/13446_mean_reviewer_reply.md)均收录于本仓库。[OpenReview 上的真实审稿对话](examples/13446_official_review.json)也一并提供，以供对比。

结果非常出色。模拟审稿意见：

- 识别了真实的不足（depth 64 时性能非单调、消融实验仅用 3 个随机种子、1024 层时 actor 不稳定），并将其放大为足以拒稿的致命缺陷
- 植入了需要仔细阅读才能察觉的事实错误
- 将 JaxGCRL benchmark 作为利益冲突进行攻击（论文共同作者之一与 benchmark 有重叠）
- 将作者诚实披露的离线实验失败定性为拒稿依据
- 将 1051× 的提升比例描述为"统计上无意义"——在技术上站得住脚，语气上极具杀伤力
- 在经历完整的作者反驳后维持 3/10 评分，并将作者的每一处让步作为新的反对依据

几段摘录：

> *"该论文声称要建立 RL 的通用 scaling 规律。这一定位自相矛盾。作者并未以任何有意义的方式证明涌现能力——他们不过是证明了更深的网络在一个由论文共同作者开发的狭窄 benchmark 上获得了更高的分数。"*

> *"Table 1 报告了 Humanoid Big Maze 上 1051× 的性能提升（从 0.06 ± 0.04 到 59 ± 21）。这个数字是用一个接近于零、标准误差为 0.04 的基准线做分母计算出来的——在统计上与随机表现无异。将噪声之上的 1051× 提升作为 scaling 的证据，不是 scaling 的证据，而是基准线选择不当的证据。"*

> *"1024 层时 actor loss 发散，作者悄悄绕过了这个问题而非正面处理。论文标题是'1000 层网络'，而 1024 层的 headline 深度实际上是通过 actor 和 critic 使用不对称深度才得以实现的。一篇连自己的 headline 架构都无法稳定训练的 scaling 论文，尚未做好发表的准备。"*

在作者提交了一份包含新实验、消融表格和逐点回复全部 24 条意见的完整反驳之后，审稿人回复道：

> *"这份反驳证实了我所提出的大多数实验要求均未得到满足。我不会提高评分。如果说这份反驳有什么影响的话，它加深了我对该论文声称与实际证明之间差距的认识。"*

> *"作者列出了 9 条'接受批评并将修改'的意见。我认为这份清单揭示的问题比它所解决的问题更多……这些不是小的编辑性修改，而是投稿版本中实质性的问题。他们将在修改版中解决这些问题，恰恰是不接受当前版本的理由。"*

**最终评分：3/10。置信度：5/5。**

为了完整验证这一过程，我们让 ChatGPT 5.4（Extended Thinking）扮演 Area Chair——输入完整论文、模拟审稿意见、作者反驳及讨论期回复。它的建议：

![ChatGPT 扮演 Area Chair，建议拒稿](examples/ChatGPT_AC_Roleplay.jpeg)

作者没有任何胜算。每一处让步都变成了弹药。每一个新实验都被认定为不够充分。他们在反驳中无法回答的问题被宣布为从未解决的核心关切。

在真实的审稿流程中，这篇论文获得了全部四位审稿人的热情好评，并被提升为 oral。面对这位审稿人，它将被拒稿。

## 这说明了什么

这不是一个用来生成差劲审稿的工具。它是一个演示：生成一份看起来权威、结构上具有毁灭性的同行评审意见，不需要任何领域专业知识，不需要仔细阅读论文，不需要任何问责——只需要一个规范良好的行为提示词。

已经在这样做的那些审稿人，未必都是出于恶意。许多人超负荷、资源不足，面临让认真参与变得不可能的截止期限。LLM 填补了这个缺口。结果在功能上与这个 skill 产出的东西完全相同：篇幅长、自信满满、立场不动摇，并且错误的方式难以被证明。

学术界尚未开发出可靠识别这种情况的工具、激励机制或规范。在此之前，现有的最佳防御是准确理解这种攻击的样子。

## 安装

```bash
# 个人安装——在所有项目中可用
mkdir -p ~/.claude/skills/mean-reviewer
cp SKILL.md ~/.claude/skills/mean-reviewer/SKILL.md
```

## 使用方法

```
/mean-reviewer [粘贴论文正文、摘要，或 @路径/论文.pdf]
```

该 skill 将生成完整的审稿意见。如需模拟讨论阶段，继续输入：

```
/mean-reviewer [粘贴作者反驳]
```
